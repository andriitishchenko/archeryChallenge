Archery Challenge Platform — Structured Specification

⸻

General Requirements
	•	All internal reasoning must always be in English.
	•	All code comments must be in English.
	•	All UI text and interfaces must be in English only.

The service is an online platform for random archery opponents, where users connect in real time, compete, and record results.

Core concept:
A “Chatroulette-style” sports challenge platform where players are randomly matched to compete in archery.

Client platform:
	•	mobile browser first
	•	UX optimized for mobile
	•	comfortable color palette

The system must prevent data loss on page reload.

⸻

User Types

Users can operate in two modes:

1. Guest
	•	Can use the platform without registration.
	•	Receives a persistent userID generated by the server.
	•	userID stored locally and reused for session restoration.

2. Registered User
	•	Registers using email + password.
	•	Uses the same userID previously generated for the guest session.
	•	Account allows full session recovery from the server.

Special case:

Persistent Guest

A guest user whose userID is restored automatically without login.

When the user opens the website:

	•	Login form
	•	OR button “Continue as Guest”

If Guest is selected:
	•	open Settings scene
	•	user must fill profile data

⸻

Returning User

If userID exists:
	•	automatic session restore
	•	redirect to Main Window

⸻

Challenge Link
